Tab Out State Color System

A simplified resource-state model: active and background tabs are both live; freed is feedback for a Tab Out action, not a separate long-term Chrome state.
Model: Live / Frozen / Sleeping + Freed feedback
State Model
Active and background should share one resource state. Active means selected; background means not selected. Both can still be loaded, run JavaScript, play audio, and hold page state. That makes both Live.
Freed is not a new Chrome state. Freed means Sleeping + just acted by Tab Out. It should be a short-term feedback layer over Sleeping, then collapse back to plain Sleeping.

Token Mapping
UI State Token Meaning How It Renders
Live --status-live: #2F7D4C Active or background; still live. Green state bar. Active/background can differ in tooltip text, not color.
Frozen --status-frozen: #6E77A8 Paused but still loaded. Blue-violet state bar and subtle row background.
Sleeping --status-sleeping: #A9A19A Discarded; reloads on selection. Taupe state bar and quiet row background.
Freed --status-freed-accent: #12879A Sleeping because Tab Out just acted. Same sleeping bar plus teal row feedback.

This keeps the state system honest: resource state is Live/Frozen/Sleeping, while Freed is an action receipt.
